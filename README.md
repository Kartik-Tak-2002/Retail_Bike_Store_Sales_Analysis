🚴‍♂️ Retail Bike Store Sales Analysis (SQL Project)
📌 Project Overview

This project focuses on analyzing a retail bike store’s sales data using SQL. It involves database creation, data cleaning, and advanced query analysis to extract meaningful business insights such as revenue trends, customer behavior, and inventory status.

🗂️ Database Structure

The project includes multiple relational tables:

Customers – customer details and location

Orders & Order Items – transaction and sales data

Products, Categories, Brands – product hierarchy

Stores & Staff – store and employee information

Stocks – inventory tracking

🧹 Data Cleaning & Preparation

Checked and handled NULL values across all tables

Identified and removed duplicate records

Trimmed unnecessary spaces for clean data consistency

Verified relationships between tables using JOIN checks

📊 Key Business Insights

The project answers important business questions such as:

💰 Total revenue generated from sales

📦 Top 10 best-selling products

🏬 Store-wise revenue performance

🌍 Sales distribution by state

👥 Top customers by spending

📈 Monthly sales trends using CTE

⚠️ Low stock products for inventory management

🧠 SQL Concepts Used

Joins (INNER, LEFT, RIGHT)

Aggregate Functions (SUM, COUNT)

Group By & Order By

Common Table Expressions (CTE)

Data Cleaning Techniques

Conditional Filtering

🚀 Key Highlights

End-to-end SQL project (Database → Cleaning → Analysis)

Real-world business problem solving

Optimized queries for performance

Insight-driven approach for decision making

📈 Outcome

This project demonstrates strong SQL skills and the ability to transform raw data into actionable insights, making it highly relevant for Data Analyst roles.
